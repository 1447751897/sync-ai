# 前端设计方向

本文档记录本项目的用户界面设计方向。当前界面是本地配置页，不是营销落地页。

## 产品形态

| 项目 | 说明 |
| --- | --- |
| 产品类型 | Codex 本地插件配置页 / 内部工具 |
| 主要场景 | 管理图片 provider 档案，选择当前启用档案，配置模型与输出路径 |
| 目标用户 | 使用 Codex 进行网页、博客、个人 IP 和视觉参考设计的个人用户 |
| 使用频率 | 需要切换 provider、模型或密钥变量时使用 |
| 主要设备 | 桌面端为主，移动端只需基本可读可用 |

## 设计关键词

整体气质：

```text
克制、清晰、本地优先、技术但不冰冷、配置导向。
```

视觉关键词：

```text
低装饰、明确分栏、紧凑表单、温暖纸感背景、绿色主操作、棕橙色点缀。
```

交互关键词：

```text
明确选择、可逆保存、显式上下文开关、可诊断状态、密钥不外露。
```

## 布局原则

当前 MVP 使用三栏布局：

```text
左侧：provider/profile 列表
中间：当前 profile 编辑表单
右侧：状态、配置摘要、后续可扩展为验证结果和最近输出
```

响应式规则：

1. 桌面端使用三栏，方便快速扫描。
2. 宽度不足时折叠为单列，上下排列。
3. 表单字段不做复杂嵌套，减少配置错误。

## 组件策略

1. provider/profile 切换使用列表按钮。
2. provider 类型使用 select。
3. 是否携带对话上下文使用 checkbox/switch。
4. 保存和新增使用明确按钮。
5. 状态区使用等宽字体展示配置摘要和错误信息。
6. 不使用嵌套卡片，不做营销页 hero，不添加与配置无关的装饰。

## 交互状态

| 状态 | 处理方式 |
| --- | --- |
| loading | 状态区显示“加载中...” |
| empty | 后续如无档案，应显示一个新增档案主操作 |
| error | 显示可操作错误信息，但不泄漏密钥值 |
| success | 保存后重新渲染当前配置摘要 |
| disabled | 后续真实 provider 字段缺失时应禁用真实生图 |
| context sharing | 默认关闭，必须由用户显式开启 |

## 色彩与字体

当前配置页采用：

1. 背景：暖米色 `#f6f3ec`。
2. 文字：深墨绿 `#16231f`。
3. 次级文字：灰绿 `#617069`。
4. 边框：低对比米灰 `#d6d0c4`。
5. 主色：深绿色 `#1e6b5c`。
6. 点缀色：棕橙 `#b45f35`。
7. 字体：系统无衬线，等宽字体用于状态面板。

## 需要避免

1. 在页面中显示或保存真实 API Key。
2. 让对话上下文隐式发送给第三方 provider。
3. 使用泛泛的“失败了”掩盖 provider 具体错误。
4. 在 MVP 期间引入重型前端框架或桌面端复杂交互。
5. 把页面做成营销落地页或个人展示页。

## 后续增强

1. 增加 provider 连通性检查按钮。
2. 增加最近输出记录。
3. 增加配置错误的字段级提示。
4. 增加真实 provider 可用性状态徽标。
